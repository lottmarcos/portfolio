# Quality Gate

This skill defines the quality bar for the portfolio project. No feature ships without passing these gates.

## Files

- `testing-checklist.md` — What to test, how to test, coverage expectations.
- `pr-review-checklist.md` — What reviewers (human or AI) must verify before approving.
- `validation-checklist.md` — Final checks before a feature is considered done.

## When to Use

- Before marking any task as complete.
- When reviewing pull requests.
- When deciding if a feature is ready to ship.
- After implementing a fix to verify it doesn't introduce regressions.
