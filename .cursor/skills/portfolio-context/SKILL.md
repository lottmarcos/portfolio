# Portfolio Context

This skill provides the foundational context about the portfolio's owner and purpose. Consult these files when making any decision that affects positioning, messaging, or technology choices.

## Files

- `profile.md` — Professional background, current role, and domain expertise.
- `positioning.md` — How the portfolio differentiates Marcos in the market.
- `achievements.md` — Key accomplishments to highlight.
- `tone.md` — Voice and communication guidelines for all content.
- `stack.md` — Technology choices and the reasoning behind each.

## When to Use

- Writing or reviewing any user-facing copy.
- Deciding which projects or skills to emphasize.
- Choosing how to present technical depth vs. breadth.
- Evaluating whether a feature aligns with the portfolio's purpose.
- Justifying or questioning technology decisions.
