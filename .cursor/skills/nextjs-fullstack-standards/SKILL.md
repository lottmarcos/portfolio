# Next.js Fullstack Standards

This skill defines the architectural standards for the portfolio's Next.js implementation. All code must align with these guidelines.

## Files

- `architecture-guidelines.md` — App Router structure, component model, routing patterns.
- `api-guidelines.md` — Server Actions, Route Handlers, input validation, error handling.
- `data-layer-guidelines.md` — Data fetching, caching strategy, revalidation patterns.
- `performance-guidelines.md` — Core Web Vitals targets, bundle budgets, optimization techniques.
- `seo-guidelines.md` — Metadata API usage, structured data, sitemap, OG images.

## When to Use

- Creating new routes or modifying existing ones.
- Deciding between Server Components, Client Components, or Server Actions.
- Implementing data fetching or caching logic.
- Optimizing performance or debugging Core Web Vitals.
- Adding or modifying metadata and SEO configuration.
