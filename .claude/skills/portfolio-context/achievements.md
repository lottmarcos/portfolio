# Achievements

Key accomplishments to highlight in the portfolio. This list should be curated, not exhaustive — prioritize depth over breadth.

## Guidelines for Selecting Achievements

1. **Impact over activity**: Show outcomes, not tasks. "Reduced API latency by 40%" > "Worked on API optimization."
2. **Decisions over implementations**: Highlight where Marcos made the call, not just where he wrote the code.
3. **Range**: Include at least one achievement from each domain (backend, frontend, architecture, leadership).
4. **Recency**: Prioritize recent work unless an older achievement is exceptionally relevant.

## Achievement Template

Each achievement should be presentable with:

- **One-line summary**: What happened and what was the impact.
- **Context**: What was the situation or constraint.
- **Contribution**: What Marcos specifically did.
- **Outcome**: Measurable result or qualitative improvement.
