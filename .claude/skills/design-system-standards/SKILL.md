# Design System Standards

This skill defines the visual and interaction standards for the portfolio. All UI work must align with these guidelines.

## Files

- `tokens.md` — Design token architecture (colors, typography, spacing, breakpoints).
- `components.md` — Component patterns, shadcn/ui customization, composition rules.
- `accessibility.md` — WCAG 2.1 AA requirements, keyboard navigation, screen reader support.
- `visual-direction.md` — Aesthetic guidelines, anti-cliche checklist, visual tuning parameters.

## When to Use

- Creating or modifying any UI component.
- Defining or adjusting design tokens.
- Reviewing visual implementations for quality.
- Verifying accessibility compliance.
- Discussing visual direction or aesthetic decisions.
